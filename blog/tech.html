Functional Programming

10.30.15

Before we can dive into Functional Programming, we first need to step back and acknowledge the existence of various programming paradigms. In particular, we'll touch upon two contrasting paradigms: Imperative vs. Declarative.

Imperative programming, also known as algorithmic programming, describes in exacting detail each step that the compiler must take to accomplish your goal. States are mutable (the simplest example of this is the re-assignment of a variable's value).

Declarative programming does not explicitly tell the compiler all the steps needed to reach your end goal. Instead, it "declares" to the compiler what you want to achieve.

To illustrate the difference between the 2 paradigms, let's take a look at a simple example. Say we want to add 1 to a number:

var number = 1;


// IMPERATIVE Solution:
var add = function(num) {
  sum = num + 1;
  return sum;
}

add(number)

// Output:
// 2


// DECLARATIVE Solution:
var add2 = function(num) {
  return num + 1;
}

add2(number)

// Output:
// 2


Now that we've got that cleared up...



Functional programming is a sub-division of declarative programming. Unsurprisingly, functions are treated as "first class citizens" in this paradigm. To sum functional code into one characteristic, it is the absence of side effects. This means that your current function doesn’t rely on outside data, and it also doesn’t change outside data. It is perfectly encapsulated within itself, and can therefore be used anywhere in your program.

Two Tenets of Functional Code:
  1. Immutable - Your data should remain unchanged. This means, for example, rather than using destructive methods, you simply create new objects on your return.
  2. Stateless - Your functions should run every time as if it were the first time, without any regard to outside variables and functions the program may have already executed. In other words, your functions should only operate on data that you passed in as arguments without relying on outside variables.
Three Functional Habits to Develop: (These may appear simple, but since we started learning how to code with Ruby, we're used to writing imperatively.)
  1. All of your functions must accept at least one argument.
  2. All of your functions must return data or another function.
  3. No loops!

Up until now, we've only grazed the surface of functional programming. I won't be going into specifics, but below you can find an overview of some notable features and benefits of functional programming:

Functional Programming Techniques:

  • Recursion
  • Pipelining
  • Currying
  • "Higher order functions" - AKA methods that perform actions on arrays in lieu of looping, such as map and reduce.
Features of a Functional Language:
  • Immutable data
  • First class functions
  • Tail call optimisation - Available as of ECMAScript 6
Advantageous Properties of Functional Programs:
  • Parallelization
  • Lazy evaluation
  • Determinism
