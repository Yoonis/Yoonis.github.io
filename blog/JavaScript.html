Looping in JavaScript vs Ruby

10.26.15

At first I thought that Ruby came with too many built-in iteration methods. Now, having gotten a glimpse of JavaScript, I lament that JavaScript only comes with 2 loops... WHILE and FOR.

Not to worry though. We can get creative with these 2 loops to build complex functions. First, it's worth noting that there are really only 2 types of loops:

  • A) Loops that run until a condition is met.
  • B) Loops that run a specified number of times.

WHILE loops fall into category A. Here is an example in JavaScript: 

var n = 1;

while (n <= 3) {
n += 1;
console.log("n = " + n);
}
// Output:
// n = 2
// n = 3
// n = 4

Here is a WHILE loop in Ruby: 
n = 1
while n <= 3 do
n += 1
p "n = #{n}" 
end
# Output:
# n = 2
# n = 3
# n = 4

Both WHILE loops continued to execute until the condition (n <= 3) was met by way of an incremental counter on n. As we can see, the concept and the output is the same, just the syntax differs.

FOR loops fall into Category B. Here is an example in JavaScript: 

for (var n = 2; n < 5; n++) {
console.log("n = " + n);
}
// Output:
// n = 2
// n = 3
// n = 4

Here is a FOR loop in Ruby: 
for n in 2..4 do
p "n = #{n}" 
end
# Output:
# n = 2
# n = 3
# n = 4

As we can see, FOR loops keep track of the incrementing for us in both Ruby and JavaScript. I actually prefer the JavaScript FOR loop because it takes 3 arguments: an incrementor, a conditional, and an action at the end of the loop. This lays out the iteration very clearly. Ruby's FOR takes an incrementor and an expression, but seems less robust. Then again, Ruby has many other options for iterating over a dataset.
