Spacing Elements, Technically...

9.28.15

If you've ever found yourself wondering what the difference between Margin, Border, and Padding is, then by all means - read on.

Take a good look at the figure below. This is called the Box Model, and it depicts the various layers of a page element.

Let's start from the inside out. We see that our site content is first encased in a layer called Padding. This is the space between the content and the border surrounding the content. You can choose to have a lot of padding, or none at all. The second layer is the Border. This is the defined edge around your content that you can choose to display in various styles (such as dashed lines, solid lines, dotted lines, etc). Or, you could choose not to display a border at all. The third layer is the Margin. This is space which you can choose to append outside of the element's border to further pad the space between this element and this element's neighbors.

Each layer has 4 sides. You can choose to set the Margin and Padding of each side with the same value. You can also set each side with a different value in one fell swoop by having 4 values to 1 property. For example, "margin 1em, 0em, 1em, 0em" will append the given space values outside of the border in a clockwise fashion: top, right, bottom, and left, respectively.

As a final tip, whenever possible, try to set the spacing values in a flexible unit, such as em or %, rather than px. This is because px is a highly specific pixel value with no flexibility or regard to screen or page size. 1em is equal to the default font size on a user's device, and % is just a percentage of the whole page container. Keeping this tip in mind will make your life a lot easier when you deal with page re-sizing and scaling.
